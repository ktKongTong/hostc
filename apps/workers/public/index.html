Cloudflare tunnel ingress for local services

Ship localhost to the edge without standing up a public origin.

hostc combines a CLI, a Worker, and a Durable Object control plane so your local HTTP and WebSocket services can be reached through a stable public URL.

How it works API surface
HTTP + WS Forwarded through one control channel
DO-backed Connection state lives in Durable Objects
Short-lived auth Signed session and connect tokens

Flow

Three pieces, one tunnel lifecycle

01

Create a tunnel session

The Worker allocates a random subdomain and returns a signed WebSocket URL for the CLI.

02

Attach the local service

The CLI maintains the control socket and relays HTTP requests or WebSocket frames to your local process.

03

Serve traffic from the edge

Public requests hit the Worker, which routes subdomain traffic into the Durable Object that owns the active tunnel connection.

CLI

Quick start shape

hostc http 3000

# local service
http://127.0.0.1:3000

# public tunnel
https://t-xxxxxxxx.hostc.dev

The CLI refreshes the signed session on a rolling window and reconnects when the Worker rotates the control socket.

API

Minimal control endpoints

POST /api/tunnels

Create a new tunnel session and return the public URL plus CLI WebSocket URL.

POST /api/tunnels/:id/refresh

Refresh the session token and rotate the short-lived connect token.

GET /api/tunnels/:id/connect

Upgrade the CLI control channel after connect-token validation.

Operational note

When a target service is offline

Browser visitors will see a dedicated service-unavailable page instead of a raw JSON error while the tunnel is disconnected or the local service is not responding.

hostc is a Worker-first tunnel project built for low-friction local publishing.
